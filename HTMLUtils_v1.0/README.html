CREDITS

This class is made by unreal4u (Camilo Sperberg). The original idea was to just output the doctype section + a nice little method to output flash content, but it grew a little to do just about everything you can do with the most common options in the HTML markup language.

ABOUT THIS CLASS

With this class, you can:

  • Create all headers needed for your HTML document, always W3C compliant.
  • Create certain W3C compliant tags with only a few options, such as embedding flash objects, creating links, including images (without worrying about the size of the image), etc.
  • Create basic lists using only PHP arrays as input.
  • It will automaticly close the body and html tag if these weren't called explicitly.
  • It can return an array that will tell you about certain problems with all files being called.
  • This class has been thoroughly tested with Xdebug so that no errors are present.
  • NOTE: It only works in PHP5!!

USAGE

This class was made with simplicity and speed in mind. However, some tags are more complicated to use than others. Please refer to usage.php for options and usage.

define('CHARSET','UTF-8'); // The charset to use.

define('JS_DIR','js/');    // The javascript's dir.

WHAT CAN THIS PACKAGE DO FOR ME?

This package can make your life a little easier because you don't have to worry about putting together a perfectly valid W3C document header all by yourself. You can even embed flash content that works with every browser I have tested (IE6, IE7, IE8, Firefox, Opera, Chrome and Safari) and is perfectly valid for the W3C Consortium. You can also add scripts (files or directly embedded), CSS, shortcut icon, meta tags, document title, head closure and body aperture, and many many options more:

  • <a>
  • <img>
  • <br>
  • <hr>
  • <p>
  • <span>
  • <blockquote>
  • <h1> - <h6>
  • <sub>
  • <sup>
  • <s>
  • <strong> (or <b>)
  • <em> (or <i>)
  • <u>
  • <pre>
  • <code>

When the object destroys itself, it can detect whether the body and html markup tags are already closed and if they aren't, the class will print the closure tags for itself. One more thing not to worry about.
